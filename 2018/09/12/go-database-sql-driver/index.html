golang库之sql/driver

  • golang
  • sql

从这一篇开始, 先给自己挖个坑, 每周一个golang官方包, 敦促自己以不停地去深入学习和了解golang这门语言, 以及官方库的设计之道

第一篇从database/sql/driver这个包说起

总述

此包目的为: 用作各个数据来扩展, 写各自驱动的接口. 所以这个包里边基本都是interface

几大类

此包中, 存在如下几个主要类

  • Driver: 驱动
  • Conn: 连接
  • Stmt:
  • Result: 执行结果
  • Rows: 查询结果
  • Tx: 事物控制

上面这几个, 如果开发中使用过数据库, 基本都应该会有影响, 它们之间的关系如下:

Driver 通过 Open 开启一个Conn
Conn 通过 Prepare 生成一个Stmt
Stmt 通过 Exec 执行一个Sql, 返回Result
Stmt 通过 Query 执行一个查询sql, 返回Rows

Conn 通过 Begin 开启一个事物, 返回Tx
Tx 通过 Commit提交一个事物, Rollback() 回滚一个事物

另外一些新接口, 例如StmtQueryContext, StmtExecContext等, 与上述中的Exec, Query方法类似, 只不过用到了Context, 以达到超时处理等效果.

官方推荐使用新接口, 老接口将会废弃

Driver驱动

Driver代表一个数据库的驱动, 是一个数据库客户端的入口, 其只有一个Open方法

用作返回一个到数据库的连接Conn, 由于database/sql包中, 做了数据库连接池的工作, 所以这里只需要返回一个新的连接即可, 而不需要考虑缓存

Conn连接

Conn代表一个客户端到数据库的连接, 通过Driver.Open开启, 一个连接只能做三件事情,

  • 准备一个sql
  • 关闭
  • 开启事物

分别对应其三个接口

其中Prepare准备一个sql, 返回Stmt, 为后续的执行和查询做准备, 并且会将Stmt与当前连接关联, 意味着, 当关闭连接的时候, 会先关闭关联的Stmt

连接关闭的时候, 由于database/sql包中做了数据库连接池的工作, 所以可以直接关闭连接, 不需要考虑缓存

Begin开启一个事物, 事物与当前连接关联, 意味着, 当关闭连接的时候, 会先关闭关联的事物, 此方法被标记位Deprecated, 除了不能使用Context控制以外,
此接口也没有提供事物隔离级别的选项(旧接口Begin使用数据库默认隔离级别, 且不允许只读), 所以有了新的接口

所以: 一个连接可以准备一个sql或开启一个事物并与连接关联, 当关闭连接的时候, 会先关闭与之关联的Stmt和事物

Stmt

代表一个准备好的sql, 可以根据参数执行sql, 所以其可做以下事情

  • 执行
  • 查询
  • 关闭
    因为执行和查询都支持参数, 而Prepare时会根据sql, 解析出其需要的参数个数, 所以Stmt还可以
  • 返回参数个数
